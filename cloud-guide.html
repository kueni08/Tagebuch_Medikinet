Cloud-Anleitung

Schnellstart für die JSONBin-Anbindung und lokale Sicherung.

Tagebuch Verlauf Cloud-Anleitung
1. Zugang in JSONBin anlegen
  1. Auf jsonbin.io anmelden und einen neuen privaten Bin erstellen.
  2. Den automatisch vergebenen Bin Identifier kopieren.
  3. Unter API Keys einen Master Key und optional einen Access Key erzeugen.

Tipp: Lege dir einen eigenen Ordner wie medikinet-tagebuch an, damit du die Daten schnell findest.

2. Zugangsdaten eintragen
  1. Scrolle auf dieser Seite zum Abschnitt Cloud-Zugangsdaten verwalten.
  2. Trage Bin-ID, Master-Key und ggf. Access-Key in die Felder ein.
  3. Speichere die Eingaben. Auf allen Seiten wird anschließend automatisch synchronisiert.

Die Daten werden verschlüsselt an JSONBin gesendet. Fällt die Cloud aus, greift automatisch die lokale Sicherung.

3. Geräte synchronisieren
  • Melde dich auf jedem Gerät einmalig mit denselben JSONBin-Daten an.
  • Nach dem Speichern lädt die App automatisch die neuesten Einträge.
  • Über den Button CSV exportieren kannst du regelmäßige Backups erstellen.

Cloud-Zugangsdaten verwalten

Prüfe oder aktualisiere hier die Zugangsdaten für dein JSONBin.

Hinweis: Die Zugangsdaten werden ausschließlich lokal im Browser gespeichert.

Tipps & Fehlerbehebung

Cloud ist nicht erreichbar

Der Status-Hinweis zeigt dir, ob gerade lokal oder in der Cloud gespeichert wird. Sollte JSONBin ausfallen, werden die Einträge lokal abgelegt und beim nächsten Online-Zugriff hochgeladen.

Zugangsdaten ändern

Nutze den Abschnitt Cloud-Zugangsdaten verwalten weiter unten auf dieser Seite. Dort kannst du die gespeicherten Daten einsehen, anpassen oder komplett löschen.

Installierbare App nutzen

Über das Browser-Menü kannst du die Seite als App installieren. Nach der Installation läuft sie offline und benötigt keinen dauerhaft entsperrten Browser.
